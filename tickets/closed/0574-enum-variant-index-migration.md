## Summary

Change `ParameterValue::Enum` to carry `u32` (variant index)
instead of `&'static str`. Migrate all nine shipping modules to
consume enum parameters through typed enums generated by
`params_enum!` (ticket 0573). Update interpreter to resolve DSL
source-text enum names to variant indices. Update test-support
macros, FFI JSON codec, and LSP completions to the new
representation.

Also drop the `&mut` from `Module::update_validated_parameters`.
After `ParameterValue::String` is removed (ticket 0576) no variant
on the update path requires destructive take: scalars are `Copy`
and `FloatBuffer` is `Arc<[f32]>` (O(1) clone). The one extant
`take_scalar` call site (`convolution_reverb/core.rs:236`) becomes
a `get_scalar` + `Arc::clone`, which is what the take costs today.
Doing this here aligns the in-process module API with the
read-only `ParamView` that Spike 5 of ADR 0045 introduces, two
spikes ahead of schedule.

Atomic: this ticket lands as one PR because the type payload
change and the trait signature change are both load-bearing.

## Notes

This is a wide but shallow change. The per-module migrations are
nearly identical in shape:

```rust
// before
if let Some(ParameterValue::Enum(v)) = params.get_scalar("mode") {
    self.mode = match v {
        "bipolar" => Mode::Bipolar,
        "unipolar_positive" => Mode::UnipolarPositive,
        ...
        _ => self.mode,
    };
}

// after
if let Some(&ParameterValue::Enum(i)) = params.get_scalar("mode") {
    if let Ok(m) = LfoMode::try_from(i) {
        self.mode = m;
    }
}
```

The `try_from` guard protects against a descriptor/caller drift
that would otherwise be impossible by construction â€” the validated
parameter path guarantees the index is in range. A debug-only
`debug_assert!` may replace the `try_from` branch once confidence
is high; defer that decision.

Migration order inside the PR: core type change + trait signature
change â†’ interpreter â†’ test-support â†’ ffi-common â†’ modules
(one-by-one) â†’ vintage â†’ ffi test plugin. The compiler drives the
change: each cargo check pass reveals the next site to fix. The
`&mut` â†’ `&` signature change and the `Enum(u32)` payload change
produce overlapping compile errors per module, so fixing them
together is more efficient than two passes.

Decision to capture in the ticket during implementation: FFI JSON
wire format for enums. Options: (a) index only (breaks readability
of manifest dumps but matches in-process representation); (b) index
with a name shadow for human-readable logs. Recommend (a) â€” the
JSON is not a user-facing artefact.
