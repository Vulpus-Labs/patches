# Patches v0.7.0 Pre-Release Report

Date: 2026-04-23 (revised 2026-04-24 after E111–E114)
Scope: code quality per crate, Module API + FFI bridge review, architecture review.

## Revision note (2026-04-24)

Since first draft, four epics landed that resolve most of the
original blockers:

- **E111** (ADR 0045 Spike 9) — ParamFrame + ArcTable fuzz,
  10k-cycle soak under alloc trap, runtime counters via taps
  (ADR 0043). Spike 9 closed.
- **E112** — `CoefRamp<K>` / `PolyCoefRamp<K,N>` primitive in
  `patches-dsp`; all biquad/SVF/ladder kernels migrated; codegen
  verified.
- **E113** (ADR 0051) — `catch_unwind` at tick boundary with
  attribution breadcrumb; `Processor::halt_info()`; player
  reload prompt + CLAP halt banner. Panicking module no longer
  takes the host down.
- **E114** (ADR 0052) — `PeriodicUpdate` trait deleted; replaced
  by `const WANTS_PERIODIC: bool` + default
  `Module::periodic_update`. Raw-pointer footgun retired.
  `ABI_VERSION = 5`.

Footgun #1 (as_periodic dangling) and the top "Must" item
(catch_unwind) are closed. Remaining v0.7.0 work in §5.

## 1. Executive summary

Patches is in good shape for a v0.7.0 cut. The workspace has a clean acyclic crate graph, allocation-safe audio thread, solid DSL/interpreter/engine seam separation, and a growing ADR corpus (50 entries) that tracks design reasoning well.

Main risks before tagging:

1. ~~**FFI ABI hardening**~~ — addressed by E113: tick-boundary
   `catch_unwind` halts engine cleanly on module panic. Silent
   frame-decode truncation in release builds still outstanding.
2. **Panic/unwrap hygiene** — hot paths in `patches-clap`,
   `patches-lsp`, `patches-engine::kernel` still panic on
   invariant violations that should be errors.
3. ~~**`as_periodic()` safety contract**~~ — retired by E114;
   trait deleted, footgun gone.

None of these block v0.7.0 as a tagged release for
internal/early-adopter use.

## 2. Per-crate code quality

### patches-core (~2.1K LOC)
Strengths: `CablePool` lifetimes enforce single-writer; `debug_assert!` over panic; clean module separation.
Weaknesses: `cable_pool.rs:157,159` panics on write-path; `cable_pool.rs:51` `as_raw_parts_mut()` relies on external FFI discipline; `params.rs` lacks comprehensive bounds validation; thin external tests (4 `compile_fail`).

### patches-dsp (~19.5K LOC)
Strengths: explicit denormal flush, strong numerical stability, hot-path allocation-free; good test files covering SVF/ladder/FFT/OLA.
Weaknesses: `halfband.rs:141` `partial_cmp().unwrap()` risks NaN escape — use `total_cmp`; `slot_deck/active_windows.rs:97,101` unwrap chains after `is_none()` checks; duplication between `halfband.rs` and `interpolator.rs` window-fill; `test_support.rs` (557 LOC) bleeds into main compile.

### patches-dsl (~8.1K LOC)
Strengths: excellent test investment (28 files, torture tests for circular refs/deep nesting); clean PEG; phase-split expander (`binding`/`composition`/`connection`/`substitute`).
Weaknesses: fat modules (`loader.rs` 506, `binding.rs` 419, `composition.rs` 463); `binding/tests.rs` tight coupling to internals; unclear ownership between `expand/substitute.rs` and `expand/binding.rs`.

### patches-interpreter (~3.3K LOC)

Weaknesses: no integration tests outside `src/tests/`; `find_module` linear scan in `descriptor_bind/mod.rs:70`; `build_from_bound` invariant (errors checked) not type-enforced; missing safe public entry-point.

### patches-modules (~21K LOC)
Strengths: 60+ inline test modules; audio-thread safe (no locks); pre-allocated state machines; good parameter docs.
Weaknesses: fat modules — `midi_arp.rs` (686), `poly_osc.rs` (564), `oscillator.rs` (527) mix waveform/sync/state; `oscillator.rs:323`, `poly_osc.rs:373,486` use `.unwrap()` on slice access (test-only but anti-pattern); `convolution_reverb/core.rs:130-150` tangles Arc threading with DSP; effects (limiter, delay variants, transient_shaper) and drum emulators have lighter coverage.

### patches-engine (~2.7K LOC)
Strengths: lock-free audio thread; cleanup thread for deferred drops; preallocated transport/MIDI poly buffers; smart adoption.
Weaknesses: `kernel.rs:93,391,393,396,398` panic on plan/param-frame mismatch (should be `Result`); `processor.rs:150,191,219` `Box::new` for deferred drops OK on cleanup path but worth documenting; `processor.rs:70` `Relaxed` ordering on `cleanup_overflow_count` — add fence note; `midi/connector.rs:38` `Mutex<EventQueueProducer>` with implicit midir threading contract.

### patches-player (~473 LOC binary)

Weaknesses: `main.rs:245` `unwrap_or_default()` silently drops invalid flags; no test coverage beyond `diagnostic_render.rs`; stdin loop blocks main thread.

### patches-io (~1.2K LOC)
Strengths: defensive bounds checks on chunk sizes; periodic flush in recorder; extension-based dispatch.
Weaknesses: 20+ unwraps in tests — precedent-setting but not unsafe; no round-trip fidelity tests; no malformed-input fuzz; no sample-rate bounds check.

### patches-clap (~?K LOC)

## 3. Module API + FFI bridge

### In-process Module trait

Two-phase construction (`prepare` → `update_validated_parameters` → `process` / `set_ports`). Works well. Notable contracts:

- ~~`as_periodic()` once-only contract~~ — replaced by
  `const WANTS_PERIODIC: bool` + default `periodic_update`
  method (E114, ADR 0052). No raw pointers in `ReadyState`.
- `PortDescriptor.index` is the **semantic** index (e.g. `in/2`), NOT the slice offset. Hand-written port resolution is easy to get backwards (`patches-core/src/modules/module_descriptor.rs:5-15`).
- No explicit `reset()` — state reset piggybacks on reconstruction. Consider whether v0.7.0 should formalize.

### FFI bridge (ADR 0045, spikes 0-9 landed)

ABI_VERSION = 5 (`patches-ffi-common/src/types.rs:8`; bumped in
E114 for `wants_periodic` / `periodic_update` vtable slots).
`FfiPluginVTable` and `HostEnv` are `repr(C)`; HostEnv layout has
a pinning test (`abi.rs:72-85`, good). Spike 9 (E111) added
ParamFrame/ArcTable fuzz targets, a 10k-cycle alloc-trap soak,
and tap-exposed runtime counters.

**Footguns:**

| # | Issue | Severity | Location |
|---|-------|----------|----------|
| 1 | ~~`as_periodic()` pointer dangling~~ — CLOSED E114 | — | — |
| 2 | Packed param/port frame layout drift between host and plugin (alignment, new `ParameterKind` variants) → silent decode truncation | Medium | `patches-ffi-common/src/param_layout.rs`, `sdk.rs:181,206` |
| 3 | Frame-decode truncation in release builds (not Err-propagated) | Medium | `patches-ffi-common/src/sdk.rs:181,206` |
| 4 | Semantic port index vs slice offset confusion | High | module_descriptor.rs |
| 5 | CablePool ping-pong — module reading output / writing input causes feedback | Medium | engine cable code |
| 6 | Plugin handle double-drop → UB | Critical | `export_plugin!` macro contract |
| 7 | `ArcTable` refcount underflow if plugin over-releases | Low (ADR 0049 RAII guard landed; remaining risk is plugin-side misuse of the raw handle API) | `patches-ffi-common/src/arc_table/` |
| 8 | ~~No `catch_unwind` on panic across FFI~~ — CLOSED E113 (tick-boundary catch, engine halt, attribution) | — | — |

**API stability for v0.7.0:**

- Freeze: `Module::process` signature, `FfiPluginVTable` layout
  (v5), `HostEnv` layout, `Module::periodic_update` signature +
  `WANTS_PERIODIC` const, `ModuleDescriptor` field names/types
  (hash depends on them).
- Likely to change post-v0.7.0: release-build decode error
  policy (should become strict); `ParameterKind` variant set.
- Deferred until out-of-tree plugins exist: ABI_VERSION
  compat policy (how old plugin binaries negotiate with newer
  hosts); strict decode-error propagation. Per-plugin
  descriptor hash is monorepo-internal — recompile on mismatch.
  (Tick-boundary panic catch already landed via E113.)

### Developer ergonomics

In-process is clean — one trait, `ModuleDescriptor` literal, inline tests. FFI adds `export_plugin!` boilerplate, a descriptor-fn that must be surfaced as a free function (so the hash symbol can call it), silent rejection on descriptor-hash mismatch between host and plugin, and no feedback on frame-decode truncation in release builds. Before encouraging third-party plugins: richer load-time diagnostics, a `patches-plugin-cli` verifier that matches descriptor bytes against a reference host, and plugin template / cookiecutter.

## 4. Architecture

### Crate graph

```
                patches-core
         ┌──────────┼──────────┐
    patches-dsl  patches-dsp  patches-ffi-common
         │          │           │
    patches-modules ◄───────────┘
         │
    patches-interpreter ─ patches-diagnostics
         │
    patches-engine ── patches-planner ── patches-ffi
         │                                   │
    patches-host ── patches-cpal        patches-ffi (dylib loader)
         │
    patches-lsp    patches-clap    patches-player
```

Verified acyclic. Boundary claims hold:
- `patches-dsl` only depends on `pest` + `patches-core`. ✓
- `patches-core` has zero audio/serialization deps. ✓
- `patches-dsp` only pulls `rtrb`. ✓

### Parallelism-readiness

`ExecutionPlan::tick()` is sequential `for_each` with 1-sample cable delay — dropping in `rayon::scope` is a contained change. No impact on `Module` trait or module implementations. Buffer layout is contiguous but **not** thread-affinity partitioned or cache-line padded; lay this groundwork before shipping parallel execution to avoid false sharing.

### Backend-agnostic core

Verified. `patches-core` has no CPAL/serde/pest/UI deps. Backends swap at `patches-host` / `patches-clap` level.

### Zero-cost descriptors

Verified — `&'static str` throughout; descriptor literals built at module compile time; O(n) name lookups with n < 20, no hashing on audio thread.

### ADR coverage

50 ADRs; recent entries include spike reports and codegen validation. Gaps worth filling before v0.7.0:
- Workspace structure / publish-flag policy
- Testing strategy (test-support feature, golden-file policy)
- Vintage module external-bundle deployment model
- ABI_VERSION compat / negotiation policy (deferred until
  out-of-tree plugins exist)

### Strengths (ranked)

1. Acyclic crate graph, clean seams — DSL/interpreter/engine/modules separated and reversible.
2. Allocation-safety baked into design (ExecutionPlan + ADR 0045 frames).
3. Parallelism path clear and contained.
4. Dense ADR trail with spike reports.
5. Backend-agnostic core, trivially portable.
6. Solid test density in DSL, LSP analysis, integration harness.

### Weaknesses / debt (ranked)

1. Panic density in LSP (341), CLAP (16 poisoned-mutex expects),
   engine kernel (5 on invariant failure). Tick catch (E113)
   covers module panics but not these hot paths.
2. Buffer layout not yet thread-affinity partitioned (prep for
   future parallel).
3. Vintage bundle external-distribution story undocumented in
   ADR.
4. Observability improving — E111 added runtime counters via
   taps (ADR 0043); still missing plan-adoption latency, pool
   utilisation, cleanup-ring saturation.
5. ~~FFI integration tests missing~~ — closed by E111 (Spike 9
   fuzz + soak).

## 5. Recommended pre-v0.7.0 work

Must (blockers for tag):

- None outstanding. FFI test-plugin smoke added to CI
  (`test-ffi-plugins.yml`) covering load/describe/process/scan,
  hash mismatch, and 10k-cycle alloc-trap under ABI v5.
- Strict frame-decode error propagation deferred with the rest
  of the external-plugin story (no out-of-tree plugins yet).

Should (cheap wins, ship if time):

- Replace `gui_state.lock().expect()` in patches-clap (still 16×
  across `extensions.rs` + `gui_vizia.rs`) with `clear_poison`
  recovery or GUI model restart.
- Replace `kernel.rs` invariant panics with `Result` — now that
  E113 halts on panic, a recoverable frame-skip is still a
  better UX than a halt banner.
- `PortDescriptor` API: helper `port_slice_offset(name: &str)`
  to reduce hand-rolled index math (footgun #4).
- LSP: reduce unwrap density on tree-sitter / diagnostic-data
  paths (341 sites, largest source of remaining panic surface).

Nice to have:

- Split fat modules: `midi_arp.rs`, `poly_osc.rs`,
  `oscillator.rs`, `loader.rs`, `binding.rs`, `composition.rs`
  into pure-core + adapter per the extract-pure-cores
  convention.
- ADR for workspace / publish / vintage-bundle.
- Document new `periodic_update` method + `WANTS_PERIODIC` in
  the manual's module-authoring page.
- Wire the E111 runtime counters into the player's status
  output (currently tap-accessible but not surfaced).

Defer to v0.8:

- Thread-affinity buffer partitioning (only if parallel
  execution lands).
- Tap/meter/scope UI (already parked).
- Plan-adoption latency + pool-utilisation metrics.
